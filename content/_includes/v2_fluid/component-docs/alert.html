Alert

Improve this Doc Alerts are a great way to provide the user with important information, prompt the user to provide information, or ask the user to confirm an action. Alerts can be thought of as a type of “floating” modal that cover only a portion of the screen. This means Alerts should only be used for quick actions like password verification, small app notifications, or options. More in depth user flows should be reserved for full screen ​[Modals](#modals)​. Alerts can be configured to include custom buttons, input fields, radio buttons, and checkboxes. _For complete usage information, check out the [Alert API docs](../api/components/alert/AlertController)._

Basic Usage

Demo Source ```typescript import { AlertController } from 'ionic-angular'; export class MyPage { constructor(public alertCtrl: AlertController) { } showAlert() { let options = { title: 'New Friend!', subTitle: 'Your friend, Obi wan Kenobi, just accepted your friend request!', buttons: [ { text: 'Cancel', handler: data => { console.log('Cancel clicked'); } }, { text: 'OK' } ] } let alert = this.alertCtrl.create(options); alert.present(); } } ```
